Dataset Construction Guidelines
Students are required to generate datasets programmatically.

Minimum Dataset Requirements
To ensure meaningful experimentation, ALL datasets must satisfy the following con-
straints unless explicitly stated otherwise.
• Minimum number of samples:
n ≥1000
• Minimum number of features:
d ≥15
• Every dataset must contain:
– at least 5 informative features,
– at least 5 noisy/irrelevant features.
• At least ONE dataset must contain:
d ≥50
features.
• At least ONE dataset must contain:
n ≥5000
samples.
• Extremely small toy datasets are NOT allowed except for:
– debugging,
– visualization,
– conceptual demonstrations.
• Direct usage of Kaggle datasets or public datasets is NOT allowed unless explicitly
mentioned.

Question 1: k-Means Clustering [30 Marks]
You must implement k-Means clustering completely from scratch.
Part A: Core Implementation
Implement:
• random centroid initialization,
• Euclidean distance computation,
• cluster assignment,
• centroid update,
• stopping criterion.
DO NOT use loops wherever vectorization is possible.
4
Part B: Adversarial Dataset Construction
Construct TWO datasets:
(a) a dataset where k-Means performs extremely well,
(b) a dataset where k-Means fails despite visually separable clusters.
Explain:
• why the failure occurs,
• which assumption of k-Means is violated,
• whether feature scaling changes the outcome.
Part C: Initialization Sensitivity
Run your implementation 20 times using different initializations.
(a) plot convergence behavior,
(b) compare final clustering costs,
(c) explain why different initializations lead to different local minima.

Part B: Correlated Features Experiment

Question 3: Decision Trees using C4.5 [40 Marks]
Part A: C4.5 Implementation
Implement a binary decision tree classifier from scratch using:
• entropy,
• information gain,
• split information,
• gain ratio.
Your implementation must support:
• recursive splitting,
• stopping criteria,
• maximum depth,
• minimum samples per split.
Part B: Gain Ratio Analysis
Using your generated datasets:
(a) compare Information Gain and Gain Ratio,
(b) identify cases where Information Gain prefers poor splits,
(c) explain why Gain Ratio improves split selection.
